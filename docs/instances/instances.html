Cloud Apps¶

File for operating on a Cloud Apps Instance.

CloudAppsInstance is the only class defined in this file.

CloudAppsInstance: Derived class from Instance Base class, representing a
cloud apps instance, and to perform operations on that instance

CloudAppsInstance:

__new__() – Method to create object based on specific cloud apps instance type

Usage¶

To add a new Instance for Cloud Apps agent, please follow these steps:

  1. Add the module for the new instance type under the location:
    /cvpysdk/instances/cloudapps, with the module name <new instance type>_instance.py (e.g. “google_instance.py”, “salesforce_instance.py”)
  2. Create a class for your instance type and inherit the CloudAppsInstance class.
  3. Add the import statement inside the __new__ method.
    NOTE: If you add the import statement at the top, it’ll cause cyclic import, and the call will fail
  4. After adding the import statement:
    • In the instance_type dict
      • Add the cloud apps instance type as the key, and the class as its value

timevalue (str) – for pointintime based restore
format: YYYY-MM-DD HH:MM:SS

sybasecreatedevice (bool) – determines whether to createdevice for sybase database restore

renamedatabases (bool) – determines whether renamedatabase option chosen for given database restore

device_options (dict(dict)) – dict of dict for each database with device rename and database rename options

Also This is dict of dict having sourcedatabasename as Key and set of another dict options as value corresponding to that source Database.

}

}
